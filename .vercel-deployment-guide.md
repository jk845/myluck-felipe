# Vercel Deployment Guide for Myluck.no

## Step 1: Deploy to Vercel

1. Go to [vercel.com](https://vercel.com) and sign in with your GitHub account
2. Click **"Add New Project"**
3. Select the repository: `jk845/myluck-felipe`
4. Configure the project:
   - **Framework Preset**: Vite
   - **Root Directory**: `./` (leave as default)
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`

5. Click **"Deploy"**

## Step 2: Add Environment Variables (if needed)

If your app uses environment variables (check `.env.example`), add them in Vercel:

1. Go to your project settings in Vercel
2. Navigate to **Settings** → **Environment Variables**
3. Add each variable from your `.env.example` file

## Step 3: Connect Your Custom Domain (myluck.no)

### In Vercel:
1. Go to your project in Vercel
2. Click **Settings** → **Domains**
3. Add your domain: `myluck.no`
4. Also add: `www.myluck.no` (optional but recommended)
5. Vercel will provide you with DNS records to configure

### In GoDaddy:
1. Log in to your GoDaddy account
2. Go to **My Products** → **DNS** for `myluck.no`
3. Add the following DNS records (Vercel will show you the exact values):

   **For the root domain (myluck.no):**
   - Type: `A`
   - Name: `@`
   - Value: `76.76.21.21` (Vercel's IP)
   - TTL: 600 seconds

   **For www subdomain (www.myluck.no):**
   - Type: `CNAME`
   - Name: `www`
   - Value: `cname.vercel-dns.com`
   - TTL: 600 seconds

4. Save the DNS records
5. Wait 5-10 minutes for DNS propagation (can take up to 48 hours in some cases)

## Step 4: Verify Deployment

1. Once DNS propagates, visit `https://myluck.no`
2. Verify SSL certificate is automatically provisioned by Vercel
3. Test all pages and functionality

## Troubleshooting

### If the domain doesn't work:
- Check DNS propagation: [whatsmydns.net](https://www.whatsmydns.net)
- Verify DNS records in GoDaddy match Vercel's requirements
- Wait longer (DNS can take time to propagate globally)

### If environment variables are missing:
- Add them in Vercel project settings
- Redeploy the project after adding variables

### If build fails:
- Check build logs in Vercel dashboard
- Ensure all dependencies are in `package.json`
- Verify Node.js version compatibility

## Additional Configuration

### Redirect www to non-www (or vice versa):
Vercel handles this automatically when you add both domains.

### Enable Analytics (optional):
1. Go to **Analytics** tab in Vercel
2. Enable Vercel Analytics for visitor insights

### Set up Preview Deployments:
Every push to a branch will create a preview URL automatically.
